## Format planów (SDD)

Każdy plik w `docs/plans/` ma format `PLAN_[feature-name]-[date].md` i zawiera 8 sekcji:

1. **Cel** — opis biznesowy funkcjonalności
2. **Zakres** — co wchodzi / nie wchodzi w zakres
3. **Wymagania funkcjonalne** — lista wymagań
4. **Wymagania niefunkcjonalne** — wydajność, bezpieczeństwo, UX
5. **Kontekst techniczny** — komponenty, API, dane
6. **Kroki implementacji** — numerowana lista kroków
7. **Kryteria akceptacji** — warunki ukończenia
8. **Testy** — unit i integracyjne
